# Gmail 通知集成指南

## 概述
系统现在支持使用租户自己的 Gmail 账号发送通知邮件，替代平台默认的邮件服务。

## 功能特性
- ✅ 租户管理员可连接自己的 Gmail 账号
- ✅ 自定义发件人名称和邮箱
- ✅ 支持 HTML 格式的邮件内容
- ✅ 自动使用 Gmail 发送所有通知邮件
- ✅ 可随时断开连接，切换回平台默认服务

## 配置步骤

### 1. 进入设置页面
管理员访问：**网站设置 → 通知设置 → Gmail 邮箱设置**

### 2. 连接 Gmail 账号
1. 点击「连接 Gmail 账号」按钮
2. 在弹出的 OAuth 窗口中登录 Google 账号
3. 授权 `gmail.send` 权限
4. 授权完成后自动返回系统

### 3. 配置发件人信息
- **发件人名称**：显示在邮件中的名称（例如：「同一物流通知中心」）
- **发件人邮箱**：可选，留空则使用 Gmail 账号的默认邮箱

### 4. 发送测试邮件
点击「发送测试邮件」按钮，验证配置是否正确。

## 技术实现

### 后端函数
- **`sendEmailViaGmail`**: 使用 Gmail API 发送邮件
  - 构建 RFC 2822 格式的 MIME 消息
  - 支持 HTML 和纯文本内容
  - 自动处理非 ASCII 字符编码（RFC 2047）

### 通知流程
```
createNotificationWithEmail
  ↓
检查租户是否配置 Gmail
  ↓
是 → sendEmailViaGmail (Gmail API)
否 → Core.SendEmail (平台默认服务)
```

### 数据库实体
**TenantEmailSettings** 实体存储配置：
- `email_provider`: "gmail" | "platform"
- `gmail_connection_enabled`: boolean
- `sender_name`: 发件人名称
- `sender_email`: 发件人邮箱
- `configured_by`: 配置者邮箱
- `configured_at`: 配置时间

## API 使用限制
- Gmail API 每日发送限额：2000 封/天
- 超出限额后会自动降级使用平台默认服务
- 建议合理配置通知模板，避免不必要的邮件发送

## 安全说明
- OAuth 令牌由 Base44 平台安全存储
- 租户只能访问自己连接的 Gmail 账号
- 系统仅请求 `gmail.send` 权限，无法读取邮件
- 所有邮件发送操作都会记录日志

## 故障排除

### 问题：连接失败
- 检查是否使用了正确的 Google 账号
- 确认已授予 `gmail.send` 权限
- 尝试清除浏览器缓存后重试

### 问题：邮件发送失败
- 检查 Gmail 账号是否欠费或被限制
- 确认未超出每日发送限额
- 查看系统日志获取详细错误信息

### 问题：收件人收到垃圾邮件警告
- 确保发件人名称专业且易识别
- 避免频繁发送相同内容的邮件
- 考虑使用自定义域名邮箱（需额外配置）

## 最佳实践
1. 使用专业的发件人名称（如「XX 物流通知中心」）
2. 定期测试邮件发送功能
3. 监控通知邮件的打开率和用户反馈
4. 重要通知建议同时开启站内通知和邮件通知
5. 避免在深夜时段发送非紧急通知

## 回滚方案
如需切换回平台默认邮件服务：
1. 进入设置页面
2. 点击「断开 Gmail 连接」
3. 系统会自动使用平台默认服务发送后续邮件
